# 📅 고교학점제 시간표 시뮬레이터

> 고교학점제 환경에서 학생 선택과목 조합의 **구조적 충돌을 분석**하고, **최적 시간표를 자동 탐색**하는 웹 시뮬레이터

🔗 **바로 사용하기:** [https://school-calendar-ashen.vercel.app](https://school-calendar-ashen.vercel.app/)

---

## ✨ 특장점

| 기능 | 설명 |
|------|------|
| **제로 설치** | 브라우저만 있으면 즉시 사용. 서버 불필요, 데이터는 로컬 저장 |
| **완전한 제약 모델링** | 교사 겹침, 분반 쌍, 교실 용량을 모두 반영한 하드 제약 검증 |
| **4종 최적화 알고리즘** | SA, Graph Coloring, CSP 백트래킹, 용량 시뮬레이션 내장 |
| **전수 시나리오 분석** | 가능한 모든 시간표 배치를 생성하여 학생 프로필 가능률 순위 산출 |
| **유연한 분반 시스템** | 어떤 과목이든 분반으로 분할 가능. 학생은 자동으로 최적 분반 배정 |
| **구조 자유 조정** | 교시 수, 교실 수를 자유롭게 변경하여 What-If 분석 가능 |
| **시수 자동 계산** | 교사별 배정 슬롯 기반 시수(1슬롯 = 3시수) 현황 표시 |

## �🖥️ 화면 구성

### 1. 시간표 배치
- 교시 × 교실 그리드에서 과목을 클릭하여 직접 배정
- 하드 제약 위반(교사 겹침, 쌍 제약) 실시간 감지 및 경고 표시
- 사이드바에서 미배정 과목 현황 확인

### 2. 시나리오 분석
- 현재 시간표에 대해 모든 학생 선택 조합(프로필)의 충돌 여부 검증
- 충돌 핫스팟(가장 많이 겹치는 과목 쌍) 시각화
- 필터링: 교양/인문 선택별 세부 조회
- 📊 충돌 불가피성 리포트: 구조적 분석 제공
- 🔬 과학 자유선택 전략: 해결 방안 제안

### 3. 전수 시간표 탐색
- 하드 제약을 만족하는 모든 유효 시간표를 생성
- 각 시간표별 시나리오 가능률 계산 후 순위 정렬
- 최적 시간표를 선택하여 즉시 적용 가능

### 4. 학생 관리
- 학생별 선택 과목 설정 및 충돌 검증
- 시드 기반 랜덤 학생 자동 생성 (재현 가능)
- 전체 학생 일괄 검증

### 5. 자동 개선 제안
- Hill Climbing 기반 두 슬롯 교환 탐색
- 충돌 감소 상위 3가지 대안 제시

### 6. 자동 시간표 생성
| 알고리즘 | 특성 |
|----------|------|
| 🔥 Simulated Annealing | 랜덤 교환으로 점진적 최적화, Greedy 초기해 옵션 |
| 🎨 Graph Coloring | DSatur/Degree/Weight 전략. 가장 빠름 |
| 🧩 CSP 백트래킹 | MRV+LCV 기반 정확 해 탐색. 해 존재 여부 증명 |
| 📊 용량 시뮬레이션 | 교실/교시 조합별 충돌 0 달성 가능 최소 자원 분석 |

### 7. 데이터 편집
- 과목 추가/편집/삭제/분반화
- 교사 추가/편집/삭제 + 시수 현황
- 선택 규칙 추가/편집/삭제
- 분반 그룹 관리
- 쌍 제약 추가/삭제
- 교실/교시 수 조정

## 🚀 사용법

### 빠른 시작 (3분)

1. [시뮬레이터 열기](https://school-calendar-ashen.vercel.app/)
2. **학생 관리** 탭 → `🎲 자동 생성` 클릭 (29명 샘플 학생 생성)
3. **자동 시간표 생성** 탭 → `🚀 SA 실행` 클릭
4. 결과 시간표를 `적용` 버튼으로 반영
5. **시나리오 분석** 탭 → `▶ 시뮬레이션 실행`으로 가능률 확인

### 커스텀 설정

1. **데이터 편집** 탭에서 학교 실정에 맞게 과목/교사/교실 설정
2. 선택 규칙(택N) 추가 — 예: "과학 5과목 중 3개 선택"
3. 필요 시 과목 분반화 — 예: 물리 → 물리1/물리2
4. 교실/교시 수 조정하여 여유 슬롯 확보
5. 자동 시간표 생성 후 전수 탐색으로 최적해 검증

### 데이터 관리

- **💾 내보내기** — 현재 설정을 JSON 파일로 저장
- **📂 가져오기** — 이전에 저장한 JSON 복원
- **↺ 초기화** — 모든 데이터를 기본값으로 리셋
- 새로고침해도 데이터 유지 (LocalStorage 자동 저장)

## 📐 제약 모델

```
┌───────────────────────────────────────────────┐
│  하드 제약 (위반 시 시간표 무효)               │
│  • 쌍 제약: 지정 과목쌍은 다른 교시 필수       │
│  • 교사 제약: 같은 교사 과목은 다른 교시 필수   │
├───────────────────────────────────────────────┤
│  소프트 제약 (최적화 대상)                     │
│  • 학생 충돌: 같은 교시 2과목 동시 수강 불가    │
│  • 목표: 전체 학생 프로필 가능률 최대화         │
└───────────────────────────────────────────────┘
```
