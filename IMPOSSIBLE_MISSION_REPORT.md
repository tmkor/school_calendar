## 1. 실행 요약 (Executive Summary)

현재 시스템 조건(20과목, 10교시, 2교실)에서 **4억 4백만 개의 유효 시간표를 전수 검증**한 결과, 모든 학생의 선택 조합을 충돌 없이 수용하는 시간표는 **수학적으로 존재하지 않습니다.**

| 지표 | 결과 |
|------|------|
| 유효 시간표 수 (모든 하드 제약 충족) | **404,790,408개** |
| 학생 프로필 수 (선택 과목 조합) | **60개** |
| 분반 조합 (프로필당 자동 최적화) | **32가지** |
| **100% 가능 시간표** | **0개** |
| 최고 가능률 | **26.7% (16/60)** |

| 중앙값 가능률 | **13.3%** |
| 최저 가능률 | **10.0% (6/60)** |

> **결론: 어떤 배치를 하더라도, 최대 26.7%의 학생 프로필만 충돌 없이 수강 가능합니다. 73.3% 이상의 학생은 구조적으로 시간표 충돌을 피할 수 없습니다.**

### 2.3 학생 선택 모델

각 학생은 다음을 이수합니다:
- **분반 5과목:** 국어·영어·수학·체육·R&E 각 1분반 (시간표에 따라 최적 배정)
- **과학 택3:** 5과목 중 3개 선택 → C(5,3) = **10가지**
- **인문/수학 택1:** 3과목 중 1개 → **3가지**
- **교양 택1:** 2과목 중 1개 → **2가지**

**총 프로필 수:** 10 × 3 × 2 = **60가지**  
**분반 최적화:** 각 프로필에 대해 2⁵=32가지 분반 조합 중 최적을 자동 탐색

## 3. 전수 분석 방법론

### 3.1 열거 방식

20개 과목을 10교시 × 2교실에 배치하는 문제는 **20개 원소를 10개 크기-2 그룹으로 분할**하는 문제입니다.

```
이론적 전체 경우의 수: 19!! = 654,729,075
하드 제약(9개) 적용 후 유효: 404,790,408개 (61.8%)
```

### 3.2 가능률 검증 방식

각 유효 시간표에 대해:
1. 60개 선택 프로필 각각에 대해
2. 32가지 분반 조합을 시도하여
3. 10과목이 모두 다른 교시에 배치 가능한 조합이 1개라도 있으면 "가능"

→ 시간표당 최대 60 × 32 = 1,920회 검증 수행  
→ 총 약 **7,772억 회** 검증 (실제로는 조기 종료 최적화 적용)

## 5. 근본 원인 분석

### 5.1 ZERO SLACK 정리

```
정리: 20과목 ÷ 2교실 = 10교시 필요 (여유 0)
      → 각 교시에 정확히 2과목이 배치됨
      → 10개의 "동시수강불가 쌍"이 필연적으로 발생
```

### 5.2 이산적 가능률의 수학적 근거

가능률이 {6, 8, 12, 16}/60으로 정확히 이산화되는 이유:
- 분반 5과목은 쌍 구조상 특정 패턴만 허용
- 과학 택3의 C(5,3)=10 조합이 교시 쌍과 상호작용하는 방식이 제한적
- 교양1/2는 같은 교시에 배치될 때 항상 무충돌 (택1이므로)

### 5.3 비둘기집 원리

```
학생의 10과목 = 5분반 + 3과학 + 1인문 + 1교양
10교시에 각 1과목씩 배정해야 함

"같은 교시" 쌍 10개 중, 학생의 2과목이 포함된 쌍이 있으면 → 충돌
60프로필의 다양성 >> 10개 쌍으로 분리 가능한 최대값
```

**분석:**

- 최적 배치를 찾으면 40%까지 가능하나, 여전히 60%의 학생이 충돌
- **한계:** 교실 1개 추가만으로는 근본 해결 불가
