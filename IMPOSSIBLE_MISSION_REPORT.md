# 고교학점제 시간표 구조적 불가능성 분석 리포트

**작성일:** 2026-05-01  
**분석 도구:** 전수 시나리오 분석 (Brute-Force Exhaustive Analysis)  

---

## 1. 실행 요약 (Executive Summary)

현재 시스템 조건(20과목, 10교시, 2교실)에서 **모든 학생의 선택 조합을 충돌 없이 수용하는 시간표는 수학적으로 존재하지 않습니다.**

| 지표 | 결과 |
|------|------|
| 유효 시간표 수 (모든 하드 제약 충족) | 31개 |
| 시나리오 프로필 수 | 60개 |
| **100% 가능 시간표** | **0개** |
| 최고 가능률 | 27% |
| 최저 가능률 | 10% |
| 가능률 중앙값 | 10~15% 구간 |

> **결론: 어떤 천재적 배치를 하더라도, 최대 27%의 학생 프로필만 충돌 없이 수강 가능합니다. 73% 이상의 학생은 구조적으로 시간표 충돌을 피할 수 없습니다.**

## 2. 시스템 파라미터

```
┌─────────────────────────────────────────────────┐
│  교시 수:     10                                 │
│  교실 수:      2 (교실 A, 교실 B)                │
│  총 슬롯:     10 × 2 = 20                       │
│  총 과목:     20                                 │
│  여유 슬롯:    0  ← 핵심 문제                    │
│  학생 이수:   10과목/인 (10교시 × 1과목)         │
└─────────────────────────────────────────────────┘
```

### 2.1 과목 구성

| 카테고리 | 과목 | 수 |
|----------|------|----|
| 국어 (분반) | 국어1, 국어2 | 2 |
| 영어 (분반) | 영어1, 영어2 | 2 |
| 수학 (분반) | 대수1, 대수2 | 2 |
| 체육 (분반) | 체육1, 체육2 | 2 |
| R&E (분반) | R&E1, R&E2 | 2 |
| 과학 (택3) | 물리, 화학, 생물, 지구과학, 정보 | 5 |
| 인문/수학 (택1) | 독서, 기하, 사회문화 | 3 |
| 교양 (택1) | 교양1, 교양2 | 2 |
| **합계** | | **20** |

### 2.2 학생 선택 모델

각 학생은 다음을 이수합니다:
- 분반 5과목: 국어·영어·수학·체육·R&E 각 1분반 (자동 배정)
- 과학 택3: 5과목 중 3개 선택 → C(5,3) = **10가지**
- 인문/수학 택1: 3과목 중 1개 → **3가지**
- 교양 택1: 2과목 중 1개 → **2가지**

**총 프로필 수:** 10 × 3 × 2 = **60가지**  
(분반 조합 2⁵=32 포함 시: 60 × 32 = **1,920가지**)

## 3. 왜 불가능한가: 수학적 증명

### 3.1 ZERO SLACK 정리

```
정리: 20과목을 10교시 × 2교실에 배치하면,
      각 교시에 정확히 2과목이 배치된다.
      같은 교시의 2과목은 어떤 학생도 동시에 수강할 수 없다.
```

이것은 **10개의 "충돌 쌍"**이 필연적으로 발생함을 의미합니다.

### 3.2 학생 관점의 제약

학생은 10과목을 10교시에 1:1 매칭해야 합니다:

```
교시1: [과목A, 과목B] → 학생은 A 또는 B 중 하나만 수강 가능
교시2: [과목C, 과목D] → 학생은 C 또는 D 중 하나만 수강 가능
...
교시10: [과목S, 과목T] → 학생은 S 또는 T 중 하나만 수강 가능
```

학생의 10과목이 무충돌이려면, **각 교시마다 정확히 1개의 본인 과목이 배치**되어야 합니다.

### 3.3 비둘기집 원리 (Pigeonhole Principle)

- 과학 5과목을 모두 다른 교시에 분산한다고 가정 → 5교시 소비
- 나머지 5교시에 15과목 배치 (교시당 2과목 + 과학과 같은 교시의 나머지 1칸)
- 학생이 택한 3과학은 3교시를 차지 → 나머지 7과목을 7교시에 매칭해야 함
- 이 7과목(분반5 + 인문1 + 교양1) 중 어느 2개라도 같은 교시에 있으면 → **충돌**

### 3.4 하드 제약의 연쇄 효과

| # | 제약 유형 | 내용 | 효과 |
|---|-----------|------|------|
| 1 | 분반 쌍 | 국어1 ≠ 국어2 | 2교시 소비 |
| 2 | 분반 쌍 | 영어1 ≠ 영어2 | 2교시 소비 |
| 3 | 분반 쌍 | 대수1 ≠ 대수2 | 2교시 소비 |
| 4 | 분반 쌍 | 체육1 ≠ 체육2 | 2교시 소비 |
| 5 | 분반 쌍 | R&E1 ≠ R&E2 | 2교시 소비 |
| 6 | 교사 겹침 | 물리 ≠ R&E1 (과학교사A) | 배치 제한 |
| 7 | 교사 겹침 | 화학 ≠ R&E2 (과학교사B) | 배치 제한 |
| 8 | 교사 겹침 | 생물 ≠ 교양1 (과학교사C) | 배치 제한 |
| 9 | 교사 겹침 | 지구과학 ≠ 교양2 (과학교사D) | 배치 제한 |

**9개 하드 제약**이 배치 자유도를 극단적으로 축소합니다.

### 3.5 조합 폭발 vs 슬롯 부족

```
학생 선택 조합:          60가지 프로필
각 프로필이 요구하는 조건: "내 10과목이 모두 다른 교시"
한 시간표에서 만족 가능:  최대 10개 교시쌍 중 호환되는 것만

이론적 상한:
  - 10개 교시쌍(같은 교시 2과목)이 존재
  - 60프로필 중 어느 것도 이 10쌍에 걸리지 않아야 무충돌
  - 하지만 60프로필의 과목 조합이 다양해서, 어떤 쌍을 만들어도 누군가는 걸림
```

## 4. 전수 분석 결과 상세

### 4.1 가능률 분포

```
가능률 구간    시간표 수    비율
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 0% ~  5%        0         0%
 5% ~ 10%        0         0%
10% ~ 15%       24        77%   ████████████████████
15% ~ 20%        0         0%
20% ~ 25%        4        13%   ███
25% ~ 30%        3        10%   ██
30% ~ 35%        0         0%
   ...
95% ~ 100%       0         0%
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
합계             31       100%
```

### 4.2 해석

- **77%의 유효 시간표**가 가능률 10~15% 구간에 집중
  - → 60명 중 6~9명만 충돌 없이 수강 가능
- **최고 성적 시간표**도 27% (60프로필 중 약 16개만 무충돌)
- **100% 가능 시간표: 0개** — 수학적으로 불존재

### 4.3 왜 31개만 유효한가?

20! (약 2.4×10¹⁸)개의 가능한 배치 중, 9개 하드 제약을 모두 만족하는 것이 31개뿐입니다. 이는 제약의 밀도가 극도로 높음을 보여줍니다.

## 5. 시각적 이해

### 5.1 이상적 시나리오 (불가능)

```
이상: 모든 60프로필이 각 교시에 정확히 1과목씩 매칭

교시1: [A, B]  ← 60프로필 중 A와 B를 동시에 택한 학생 = 0명
교시2: [C, D]  ← 60프로필 중 C와 D를 동시에 택한 학생 = 0명
...
교시10:[S, T]  ← 60프로필 중 S와 T를 동시에 택한 학생 = 0명

필요 조건: 10개 교시쌍 모두가 어떤 프로필에도 포함되지 않아야 함
현실: 60프로필이 커버하는 과목쌍이 너무 다양 → 불가능
```

### 5.2 최선 시나리오 (27%)

```
최선의 배치에서도:
  - 무충돌: ~16/60 프로필 (27%)
  - 충돌:   ~44/60 프로필 (73%)

→ 73%의 학생 유형이 시간표와 맞지 않아 
  "원하는 과목을 모두 수강할 수 없음"
```

## 8. 최종 결론

> **이것은 알고리즘의 실패가 아니라, 물리적 자원(교실×교시)의 절대적 부족입니다.**
>
> 20과목을 20슬롯에 빈틈없이 채우는 구조에서는, 어떤 최적화 알고리즘(SA, CSP, 유전 알고리즘 등)을 사용해도 100% 무충돌 시간표를 만들 수 없습니다.
>
> 이는 NP-hard 문제의 난이도가 아닌, **해 자체가 존재하지 않는** infeasibility 문제입니다.
